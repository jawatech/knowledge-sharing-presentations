ES5 Strict mode: Syntax errors

  1. Octal syntax var n = 023;
  2. with statement
  3. Using delete on a variable name delete myVariable;
  4. Using eval or arguments as variable or function argument name
  5. Using one of the newly reserved keywords: implements, interface, let, package, private, protected, public, static, and yield
  6. Declaring function in blocks if (a<b){ function f(){} }
  7. Obvious errors:
    • Declaring twice the same name for a property name in an object literal {a: 1, b: 3, a: 7}
    • Declaring two function arguments with the same name function f(a, b, b){}

ES5 Strict mode: Runtime errors

  1. Setting a value to an undeclared variable
  2. Poisoned arguments and function properties
  3. Trying to delete a non-configurable property
Reference

ES5 Strict mode: Semantic differences

  • this in function calls
    In function calls like f(), the this value was the global object. In strict mode, it is now undefined
  • arguments doesn't alias named function arguments
    In non-strict mode, modifying a value in the arguments object modifies the corresponding named argument.
    In strict mode, the arguments object is created and initialized with the same values than the named arguments, but changes to either the arguments object or the named arguments aren't reflected in one another.
  • Change to eval
    In strict mode code, eval doesn't create a new variable in the scope from which it was called.

Object descriptors

  1. Data descriptors – a property that has a value, which may or may not be writable
  2. Accessor descriptors – a property described by a getter-setter pair of functions

A descriptor must be one of these two flavors; it cannot be both.
