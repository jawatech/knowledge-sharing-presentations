Modern Frontend Part 2

The problem

  • Web sites are turning into Web apps
  • Code complexity grows as the app gets bigger
  • It becomes harder to manage the application state and update the view

Static pages

  • Page reloads after every user action
  • Cons: losing state after reload, speed, UX

AJAX

  • Parts of the page are updated via background requests
  • SPA (Single Page Application) approach

Document Object Model

  • The DOM was designed for static content, not frequent updates
  • Updating the DOM is very slow

DOM update flow

  1. Load data from the server
  2. Render the DOM tree using the data
  3. Handle user input to update the data
  4. Change the DOM to match the updated data *

Step 4 is usually the hardest and the most error-prone to implement, when the web app becomes more complex

What if we could rerender the whole app like in the good old 90's?

React.js

A view library introduced by Facebook in 2013

  • Declarative
  • Component based
  • Rerender on every state change
  • Virtual DOM
  • One way data flow

Virtual DOM

Building components, not templates

  • Templates are often underpowered
  • Templates reinvent concepts that already exist in JavaScript
  • Templates and UI logic are highly coupled
  • Templates separate technologies, not concerns
